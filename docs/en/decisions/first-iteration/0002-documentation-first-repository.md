VSlices documentation will start as a dedicated documentation repository.

The purpose of this repository is not to present VSlices as a finished suite, but to define and document the four VSlices products while they are validated through Domus Orbis.

The documentation will be organized around:
- Method
- Design
- Docs Standard
- Framework

But it will also include:
- Start Here
- Guides
- Domus Orbis notes
- Architectural/product decisions

Read the Docs will be used initially to avoid over-investing in branding, visual identity, or website design before the product boundaries are stable.
