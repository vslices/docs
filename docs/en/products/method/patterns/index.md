# Method Patterns

Method Patterns describe recurring working decisions inside VSlices Method.

They help teams preserve continuity while moving through uncertainty, collaboration, documentation, implementation and feedback.

A pattern is not a required process. It is a reusable way to reason about a situation that appears often during real work.

## Purpose

VSlices Method patterns help answer questions such as:

* How should we choose a design modality?
* When should we change modality?
* How do people collaborate without losing intent?
* How does feedback return to the next iteration?
* How do we avoid turning documentation into ceremony?
* How do we preserve continuity while work keeps moving?

Patterns exist to make repeated decisions easier to recognize. They should clarify judgement, not replace it.

## How to use patterns

Use a pattern when the current situation resembles the problem it describes.

Do not apply a pattern only because it exists. A pattern is useful when it helps the team:

* name the situation
* understand the risk
* choose a smaller next step
* preserve relevant knowledge
* avoid accidental complexity
* continue learning

If a pattern creates more ceremony than clarity, do not use it.

## Guiding principle

Method Patterns follow the same principle as VSlices Method:

> Use the smallest useful working structure that preserves the knowledge future work depends on.

A pattern should make work clearer, safer or more continuous. It should not become a rule that teams follow without understanding why.
