# Future adoption paths

> **This page is a work in progress**.

VSlices is still being defined and validated through real software design and implementation work. Because of that, this documentation does not yet provide official adoption paths.

Defining adoption paths too early would create a false sense of maturity. VSlices is intended to be progressive and composable, but the recommended ways to adopt it should emerge from real usage, not from assumptions.

## What this page will eventually describe

This page may eventually describe how to adopt VSlices progressively through different entry points, such as:

- starting from **VSlices Design**
- starting from **VSlices Docs Standard**
- starting from **VSlices Framework**
- using VSlices only as a documentation and design approach
- using VSlices in an existing .NET project
- combining the four products over time

## Current recommendation

For now, do not treat VSlices as a complete suite with official adoption paths.

Start by reading:

- [What is VSlices?](what-is-vslices.md)
- [Why VSlices exists](why-vslices-exists.md)
- [Suite overview](suite-overview.md)
