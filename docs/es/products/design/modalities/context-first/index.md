# Context-First

Context-First se usa cuando el dominio todavía es amplio, incierto, fragmentado o complejo organizacionalmente.

La prioridad es entender el entorno de negocio antes de acotar la solución.

## Cuándo usarlo

Usa Context-First cuando:

- el contexto de negocio no está claro
- distintos actores usan lenguaje diferente
- los límites aún no son visibles
- hay muchos sistemas, áreas o responsabilidades implicadas
- la solución no debería decidirse todavía

## Qué enfatiza

Context-First enfatiza:

- actors
- responsibilities
- boundaries
- language
- processes
- surrounding conditions

## Resultado esperado

El objetivo no es obtener la solución final. El objetivo es construir suficiente entendimiento compartido para que las decisiones posteriores sean más seguras.
