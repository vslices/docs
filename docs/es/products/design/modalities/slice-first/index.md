# Slice-First

Slice-First se usa cuando el equipo ya entiende lo suficiente como para construir una pequeña behavior slice útil y aprender de ella.

La prioridad es construir una pieza acotada, validar supuestos y refinar el entendimiento mediante feedback real.

## Cuándo usarlo

Usa Slice-First cuando:

- el equipo ya tiene suficiente claridad
- una pequeña implementación puede enseñar algo útil
- el feedback de la implementación es valioso ahora
- la solución puede validarse de forma segura por partes

## Qué enfatiza

Slice-First enfatiza:

- building
- validation
- feedback
- narrow scope
- learning from implementation

## Resultado esperado

El objetivo es producir una slice pequeña pero útil que ayude a confirmar, corregir o refinar la comprensión del dominio.
