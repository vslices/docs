# Problem-First

Problem-First se usa cuando el dolor es visible, pero la causa subyacente o la solución correcta todavía no están claras.

La prioridad es clarificar qué problema real se está resolviendo antes de decidir qué construir.

## Cuándo usarlo

Usa Problem-First cuando:

- existe una tensión clara
- el equipo conoce el dolor, pero no su causa
- hay varias soluciones posibles
- la solución no debería asumirse demasiado pronto
- el contexto alrededor del problema todavía importa

## Qué enfatiza

Problem-First enfatiza:

- the problem
- impact
- affected people
- constraints
- alternatives
- candidate capabilities

## Resultado esperado

El objetivo es llegar a un entendimiento suficientemente claro del problema como para tomar una decisión de diseño útil, sin saltar directamente a una solución prematura.
