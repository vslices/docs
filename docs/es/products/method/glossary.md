# Glosario de VSlices Method

Este glosario define términos usados por VSlices Method.

Method se centra en continuidad, coordinación, preservación del conocimiento y relación entre trabajo real y documentación.

## Conceptos de Method

- __Continuity path__: la secuencia de conocimiento, decisiones y artefactos que permite que el trabajo futuro siga conectado con el razonamiento original.

- __Context type__: la forma general del escenario en el que el equipo está trabajando, como un contexto todavía incierto, un problema visible o una slice suficientemente clara para construir.

- __Preservation__: el acto de registrar conocimiento de forma que siga siendo útil más adelante para decisiones, implementación o validación.

- __Iteration__: un ciclo de trabajo en el que el entendimiento, la documentación, el diseño, la implementación y la validación se retroalimentan entre sí.

- __Working agreement__: una decisión compartida sobre cómo un equipo coordina documentación, diseño e implementación durante el trabajo real.

## Conceptos de apoyo

- __Reasoning mode__: una manera de enfocar el pensamiento según el tipo de incertidumbre actual.

- __Design input__: el conocimiento que alimenta una decisión de diseño.

- __Documentation input__: el conocimiento que debe conservarse para trabajo futuro.

- __Implementation feedback__: el aprendizaje que vuelve desde la entrega, la validación o el uso real.

- __Learning loop__: el patrón por el cual el trabajo producido hoy modifica el entendimiento que se usará mañana.

## Relación entre conceptos

Method trata el trabajo como una cadena conectada:

```text
Context type
-> Reasoning mode
-> Design input
-> Documentation input
-> Implementation feedback
-> Learning loop
```

La cadena no es estricta ni fija. Solo describe cómo Method piensa sobre la continuidad del trabajo.
