# Cambio de modalidad

El cambio de modalidad permite que el equipo adapte su enfoque cuando cambia la incertidumbre.

Un equipo puede comenzar con Context-First, pasar a Problem-First y luego usar Slice-First cuando el contexto ya sea suficientemente claro.

## Idea central

La modalidad no define al equipo.

La modalidad define el mejor enfoque para la iteración actual.
