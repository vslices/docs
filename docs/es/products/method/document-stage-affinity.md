# Afinidad entre documento y etapa

Este documento describe cómo ciertos tipos de documentos suelen encajar mejor con distintas etapas del trabajo.

El objetivo no es imponer una taxonomía rígida. El objetivo es ayudar a preservar el conocimiento correcto en el momento correcto.

## Afinidad general

- Context documents suelen apoyar Understanding y Contextualizing.
- Process documents suelen apoyar Contextualizing y Planning.
- Use case documents suelen apoyar Planning y Building.
- Decision records suelen apoyar Planning y Validation.
- Validation notes suelen apoyar Building y Learning.
- Support notes suelen apoyar continuidad operativa y trabajo futuro.

## Idea central

No todos los documentos necesitan crearse en todas las iteraciones.

La pregunta útil es:

> ¿Qué tipo de conocimiento debe preservarse para que el siguiente paso siga teniendo sentido?

Cuando un documento captura el conocimiento correcto, ayuda a que el ciclo de trabajo no pierda continuidad.
