# VSlices Method

VSlices Method ayuda a los equipos a decidir cómo trabajar con el conocimiento del dominio, la documentación, el diseño, la arquitectura y la implementación como partes conectadas de un mismo ciclo de vida.

Method no prescribe un proceso rígido. Proporciona una forma de mantener continuidad mientras el equipo avanza por escenarios con distinta incertidumbre.

La pregunta central de Method es:

> ¿Qué necesita preservarse ahora para que el trabajo futuro siga teniendo sentido?

Method se usa mejor junto con VSlices Design y VSlices Docs Standard. Design ayuda a elegir la modalidad de razonamiento correcta. Docs Standard ayuda a preservar el conocimiento importante. Method ayuda a decidir cómo se relacionan esas piezas durante el trabajo real.

## Qué hace Method

VSlices Method ayuda a los equipos a:

- identificar el tipo de contexto en el que están trabajando
- decidir qué modalidad de diseño encaja mejor
- preservar el entendimiento en documentación útil
- evitar estructuras demasiado tempranas o demasiado tardías
- revisar decisiones a medida que aparece nueva información
- conectar aprendizaje, implementación y evolución

Method se centra en la continuidad. No intenta reemplazar el razonamiento de negocio ni la disciplina de ingeniería.

## Flujo de trabajo

Method suele apoyar un flujo de trabajo que se mueve entre entendimiento, documentación, diseño, implementación y validación.

```text
Understanding -> Documentation -> Design -> Implementation -> Validation -> Learning
```

Ese flujo no siempre ocurre de forma lineal. A veces el equipo vuelve a entender mejor el contexto después de validar una slice. A veces la documentación cambia antes que el código. A veces una decisión necesita revisarse antes de seguir construyendo.

Method existe para hacer visible ese movimiento y ayudar al equipo a mantener la continuidad.

## Relación con los otros productos

Method conecta con los otros productos de la suite:

- VSlices Design ayuda a elegir cómo razonar en la situación actual.
- VSlices Docs Standard ayuda a preservar el conocimiento descubierto.
- VSlices Framework puede reflejar el resultado en software ejecutable cuando sea apropiado.

Method no obliga al uso de VSlices Framework. Tampoco exige una arquitectura específica.

Su enfoque es la coordinación del trabajo y la preservación del conocimiento.
