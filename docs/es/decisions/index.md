# Decisiones

Las decisiones preservan elecciones significativas tomadas mientras VSlices evoluciona.

Esta sección existe porque VSlices no es solo un conjunto de conclusiones finales. También es el resultado de tradeoffs, restricciones, ideas aplazadas, opciones rechazadas y lecciones aprendidas mediante diseño, documentación, implementación y validación. Una decisión puede afectar:

* VSlices Method
* VSlices Design
* VSlices Docs Standard
* VSlices Framework
* la estructura de la documentación
* la evolución de la suite
* la forma en que las ideas se validan antes de convertirse en guía oficial

## Por qué se documentan las decisiones

Los productos de software no evolucionan solo a través de features.

También evolucionan a través de decisiones. Algunas son pequeñas y temporales. Otras moldean la dirección de toda la suite. Cuando esas decisiones no se preservan, el trabajo futuro puede perder el contexto que las hacía razonables.

Los registros de decisión ayudan a preservar:

* qué se decidió
* por qué importaba la decisión
* qué alternativas se consideraron
* qué tradeoffs se aceptaron
* qué riesgos siguen presentes
* qué puede cambiar en el futuro

El objetivo no es hacer permanente cada pensamiento. Es evitar que el razonamiento importante desaparezca.

## Cómo leer estas decisiones

Las decisiones deben leerse como registros históricos y contextuales.

Una decisión explica lo que tenía sentido en un momento específico, con la información, las restricciones y las prioridades disponibles en ese momento.

- Algunas decisiones pueden seguir estables.
- Algunas pueden revisarse.
- Algunas pueden reemplazarse cuando aparece nueva evidencia.

Eso es lo esperado. VSlices favorece la arquitectura progresiva y la documentación progresiva, así que las decisiones deben evolucionar cuando cambian el dominio, el producto o la evidencia.

## Qué pertenece aquí

Esta sección debe preservar decisiones que tengan consecuencias significativas. Una decisión puede pertenecer aquí cuando afecta:

* alcance del producto
* estructura de la documentación
* dirección arquitectónica
* estrategia de implementación
* estrategia de validación
* nomenclatura
* comunicación pública
* compatibilidad futura

No toda preferencia necesita un registro de decisión.

Si una elección no tiene un tradeoff, consecuencia, riesgo o impacto futuro significativos, probablemente no necesita documentarse aquí.

## Relación con VSlices Docs Standard

Esta sección usa la idea de Registro de decisión (_Decision Record_ en inglés) de VSlices Docs Standard.

Un Registro de decisión es útil cuando una elección tiene suficiente peso como para que el trabajo futuro pueda depender de entender por qué se tomó. La misma idea puede usarse dentro de proyectos externos que adopten VSlices Docs Standard.

## Estado actual

Las decisiones se están documentando progresivamente. Algunas decisiones pueden describir direcciones estables.

Otras pueden describir elecciones temporales hechas para mantener el enfoque de la iteración actual.

Por ahora, esta sección debe leerse como un registro de cómo se está dando forma a VSlices, no como una historia completa de todas las discusiones detrás de él.
