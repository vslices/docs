# ¿Cómo empezar?

El objetivo de esta sección es ayudarte a entender qué es VSlices, por qué existe, cómo se relacionan sus productos entre sí y qué se está validando intencionalmente todavía.

## Orden de lectura recomendado

Si eres nuevo en VSlices, comienza con estas páginas:

1. [¿Qué es VSlices?](what-is-vslices.md)
2. [Por qué existe VSlices](why-vslices-exists.md)
3. [Vista general de la suite](suite-overview.md)
4. [Rutas de adopción futuras](adoption-paths.md)

Este orden introduce VSlices desde la idea general antes de pasar a la documentación específica de producto.

## Lecturas fundamentales

Después de la vista general inicial, estas páginas explican el razonamiento más profundo detrás de VSlices:

- [La continuidad del conocimiento](../philosophy/continuity-of-knowledge.md) explica por qué VSlices trata el software como un problema de continuidad entre conocimiento de negocio, documentación, arquitectura, implementación y evolución.
- [VSlices y las metodologías de desarrollo](../comparisons/development-methodologies.md) explica cómo VSlices se relaciona con Agile, Scrum, Extreme Programming, Lean, Domain-Driven Design, Waterfall y otros enfoques de desarrollo.

Estas páginas no son obligatorias antes de leer la documentación de producto, pero ayudan a aclarar qué intenta preservar VSlices y en qué se diferencia de las metodologías existentes.

## Si buscas algo específico

- Si quieres entender el panorama completo, sigue el orden de lectura recomendado.
- Si quieres entender la base filosófica de VSlices, lee [La continuidad del conocimiento](../philosophy/continuity-of-knowledge.md).
- Si quieres entender los límites de VSlices, lee [¿Qué es VSlices?](what-is-vslices.md#what-vslices-is-not).
- Si quieres comparar VSlices con otras metodologías de desarrollo, lee [VSlices y las metodologías de desarrollo](../comparisons/development-methodologies.md).
- Si quieres diseñar software antes de la implementación, lee [VSlices Design](../products/design/index.md).
- Si quieres documentar un sistema, lee [VSlices Docs Standard](../products/docs-standard/index.md).
- Si quieres construir con .NET, empieza por [VSlices Framework](../products/framework/index.md).
- Si quieres entender cómo VSlices se valida mediante trabajo real, lee la [sección Alive Lab](../alive-lab/index.md).
- Si quieres entender las decisiones metodológicas y de producto de VSlices, ve a la [sección Decisions](../decisions/index.md).

## Estado actual

VSlices todavía se está definiendo y validando.

Algunas páginas están marcadas intencionalmente como trabajo en progreso. Otras áreas, como las rutas de adopción, se posponen intencionalmente hasta que exista suficiente validación a partir del trabajo real.

Se espera que esta documentación evolucione a medida que los productos se vuelven más claros mediante trabajo real de diseño, documentación, validación e implementación.
