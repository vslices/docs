# Rutas de adopción futuras

> __Esta página está en progreso__.

VSlices todavía se está definiendo y validando mediante trabajo real de diseño e implementación de software. Debido a eso, esta documentación aún no proporciona rutas de adopción oficiales.

Definir rutas de adopción demasiado pronto crearía una falsa sensación de madurez. VSlices está pensado para ser progresivo y componible, pero las formas recomendadas de adoptarlo deben surgir del uso real, no de suposiciones.

## Qué describirá esta página finalmente

Esta página puede describir finalmente cómo adoptar VSlices progresivamente a través de diferentes puntos de entrada, como:

- comenzar desde __VSlices Design__
- comenzar desde __VSlices Docs Standard__
- comenzar desde __VSlices Framework__
- usar VSlices solo como un enfoque de documentación y diseño
- usar VSlices en un proyecto .NET existente
- combinar los cuatro productos con el tiempo
