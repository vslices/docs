# Notas de campo

Las Notas de campo es una colección de lecciones, observaciones, errores, realizaciones y patrones extraídos de proyectos reales de software y experiencia profesional.

A diferencia del resto de la suite VSlices, las Notas de campo no se centran en:

- arquitectura
- implementación
- diseño
- documentación
- proceso

En cambio, se centran en las realidades que surgen cuando personas, equipos, organizaciones y software interactúan con el tiempo.

## Por qué existen las Notas de campo

El software lo construyen personas.

- Las personas toman decisiones.
- Las personas cometen errores.
- Las personas llevan supuestos, ambiciones, miedos, fortalezas y limitaciones a cada proyecto que tocan.

Muchos de los fallos más costosos observados a lo largo del desarrollo de software no son causados por la tecnología. Surgen de:

- desalineación
- expectativas irreales
- mala comunicación
- supuestos invisibles
- mentoría descuidada
- fricción organizacional
- batallas personales de las que nadie habla hasta que se vuelven visibles

Las Notas de campo existen para preservar las lecciones ocultas dentro de esas experiencias.

## Qué no son las Notas de campo

Las Notas de campo no son:

- psicología
- terapia
- coaching
- consultoría de gestión
- doctrina organizacional
- verdad universal

Las notas representan lecciones aprendidas de situaciones específicas. Deben tratarse como reflexiones de ingeniería y no como prescripciones.

## Relación con la suite VSlices

El resto de la suite VSlices se enfoca en preservar la continuidad entre:

- descubrimiento
- documentación
- arquitectura
- implementación
- evolución

Las Notas de campo extienden esa filosofía preservando la continuidad de la experiencia.

Su objetivo es asegurar que las lecciones valiosas aprendidas a través de proyectos reales no se pierdan con el tiempo.

## Una nota sobre la verdad

Toda Nota de campo comienza con una realización y la mayoría de las realizaciones comienzan con un supuesto incorrecto.

Por esta razón, las Notas de campo no deben leerse como declaraciones de verdad, sino como correcciones a un malentendido previo.

Muchas de las lecciones documentadas aquí pueden resumirse como:

> Creía que el problema era una cosa.
>
> Con el tiempo, descubrí que era otra cosa.

Esos descubrimientos son la razón por la que existe esta colección.
