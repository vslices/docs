# Modelado relacional surreal

## Estado

Exploratorio

## Motivación

La construcción de los números surrealistas sugiere que sistemas más grandes pueden emerger de definiciones relacionales.

Los números surrealistas no reemplazan a los números reales. En cambio, los contienen.

La estructura mayor preserva la menor mientras extiende sus capacidades expresivas. Esta observación puede tener implicaciones para el modelado de software.

## Idea central

Las descripciones tradicionales de ingeniería a menudo se centran en objetos. Ejemplos:

* Usuario
* Orden
* Factura
* Producto

Un enfoque relacional se centra en las relaciones entre conceptos. Ejemplos:

Customer
→ deja
→ Orden

Orden
→ genera
→ Factura

Factura
→ Cancela
→ Deuda

La hipótesis no es que el modelado basado en objetos sea incorrecto. Es que el modelado relacional puede representar un espacio más general que contiene las descripciones orientadas a objetos como subconjunto.

## Posible conexión con VSlices

Los conceptos actuales de VSlices ya pueden ser parcialmente relacionales. Ejemplos:

Feature
→ se expone mediante
→ Integrator

Integrator
→ implementados por
→ Adapter

Documentation
→ preserva
→ conocimiento

Method
→ guia
→ implementación

Design
→ estructura
→ Conocimiento

El modelo existente sigue siendo válido; la interpretación relacional simplemente proporciona una capa adicional.
