# Getting started

> This section is still a work in progress.

VSlices Framework is currently being defined and validated through real implementation work.

Because of that, this documentation does not yet provide a complete official getting started experience.

The current focus of the Framework is to stabilize its foundational concepts before defining long-term onboarding flows, batteries, adapters, and development conventions.

## Current state

At the current stage, the Framework already includes foundational ideas and implementation direction around concepts such as:

- flows
- features
- explicit errors
- runtime requirements
- domain types
- algebraic types
- traits
- capabilities
- functional composition

However, many APIs, patterns, and package boundaries are still evolving.

The Framework should currently be considered experimental.

## What to expect from this section

This section will eventually contain:

- installation guides
- first application examples
- project structure recommendations
- feature implementation walkthroughs
- domain modeling examples
- integration examples
- testing examples
- batteries and adapters usage
- migration and adoption guidance

These guides will be written progressively as the Framework becomes more stable through real usage.

## Current recommendation

For now, the best way to understand the Framework is to explore:

- the core concepts
- the design patterns
- the Alive Lab
- Domus Orbis
- and the recorded decisions behind the Framework evolution

The current implementation work should be treated as exploratory engineering work, not as a finalized framework experience.
